# Memory Persistence — 会话记忆持久化指南

## 概述

Claude Code 每次会话独立，上下文在会话结束后丢失。本机制通过会话生命周期钩子，将关键上下文持久化到 `.claude/memory/` 目录，实现跨会话知识沉淀。

## 目录结构

```
.claude/memory/
├── project-context.md       # 项目上下文（架构决策、技术选型、核心约束）
├── active-tasks.md          # 当前进行中的任务和状态
├── decisions.md             # 关键技术决策记录
├── patterns.md              # 代码模式和惯例
├── issues.md                # 已知问题和排坑经验
└── contacts.md              # 关键模块负责人
```

## 记忆分类与优先级

| 优先级 | 分类 | 示例 |
|--------|------|------|
| P0-必记 | 关键决策 | "选择 WebSocket 而非轮询，因为需要实时推送" |
| P0-必记 | 踩坑经验 | "MyBatis-Plus Join 查询需要手动指定 resultType" |
| P1-应记 | 代码模式 | "所有 Client 类使用 WebFlux 而非 RestTemplate" |
| P1-应记 | 任务状态 | "用户模块重构完成 3/7，下一步处理权限校验" |
| P2-可记 | 偏好发现 | "该项目偏好 Builder 模式构造复杂对象" |
| 不记 | 临时信息 | 调试日志、临时变量、探索性尝试 |

## 记忆文件格式

```markdown
# [分类名称]

> 最后更新: YYYY-MM-DD HH:mm

## [主题]
- **日期**: YYYY-MM-DD
- **上下文**: [什么场景下产生的]
- **内容**: [具体信息]
- **关联文件**: [相关文件路径]
```

## 保存触发条件

仅当以下情况发生时才写入：
1. 做出了新的技术决策
2. 发现并解决了非显而易见的问题
3. 发现了项目特有的代码模式
4. 任务进度发生变化
5. 用户明确要求记住某些信息

## 保存原则

- **增量更新**: 只追加或更新变化的部分
- **去重**: 避免重复记录
- **精简**: 每条记忆 3 行以内
- **可操作**: 对未来会话有实际参考价值
- **有上下文**: 记录产生背景

## 记忆清理策略

| 触发条件 | 清理动作 |
|---------|---------|
| 任务已完成 | 从 active-tasks.md 移除，归档决策到 decisions.md |
| 文件超过 100 行 | 合并相似条目，删除过时信息 |
| 问题已修复 | 标记为已解决，保留经验部分 |
| 架构重大变更 | 更新 project-context.md，清理不适用的模式 |

## 与其他机制的配合

| 机制 | 配合方式 |
|------|---------|
| DESIGN.md | 详细版在 DESIGN.md，memory 只保留一句话摘要 |
| TODO.md | 进度在 TODO.md，memory 只记概况和阻塞点 |
| rules/ | rules 是硬性规则，memory 记录规则背后的"为什么" |
| strategic-compact | 压缩时优先保留 memory 中 P0 级别信息 |
