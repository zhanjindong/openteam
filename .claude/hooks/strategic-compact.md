# Strategic Compact — 上下文压缩协议指南

## 概述

当对话过长时系统会自动压缩历史消息。本协议定义结构化压缩策略，确保关键信息被保留，避免"压缩失忆"。

## 压缩优先级模型

从 L4 开始丢弃，逐级向上：

| 级别 | 策略 | 说明 |
|------|------|------|
| **L1 — 绝不丢弃** | 完整保留 | 任务目标、已确认设计方案、关键约束、已修改文件清单、当前进度、未解决阻塞 |
| **L2 — 尽量保留** | 压缩为摘要 | diff→"文件X: 新增方法A"、备选方案→"选了B因为..."、调试发现→"根因是XX" |
| **L3 — 可压缩** | 一句话或丢弃 | 完整文件内容、命令输出、搜索过程、多轮讨论中间消息 |
| **L4 — 可丢弃** | 直接丢弃 | 问候语、工具调用原始 JSON、重复错误信息、已放弃方案的讨论 |

## 压缩触发时机

1. **对话超过 30 轮**: 主动整理上下文摘要
2. **切换任务**: 旧任务压缩为摘要
3. **关键里程碑**: 完成 TODO.md 一个阶段后压缩该阶段过程

## 压缩执行步骤

1. 将 L1 信息完整列出
2. 将 L2 信息压缩为摘要
3. 将有价值的发现写入 `.claude/memory/`
4. 输出压缩摘要

## 压缩摘要模板

```markdown
## 上下文压缩摘要

### 当前任务
- **目标**: [一句话描述]
- **进度**: [M/N] — 参见 TODO.md
- **当前阶段**: [正在做什么]

### 已完成的工作
- [文件A]: [做了什么改动]

### 关键决策
- [决策1]: [选了什么，为什么]

### 待解决的问题
- [问题1]: [状态和下一步]

### 关键约束（用户明确要求）
- [约束1]
```

## 与外部文件的配合

| 外部文件 | 压缩时的作用 |
|---------|-------------|
| `DESIGN.md` | 设计决策的"真相源"，压缩时只需引用不需重复 |
| `TODO.md` | 任务进度的"真相源"，只需记录当前步骤 |
| `.claude/memory/` | 跨会话持久化层，压缩时写入有价值的发现 |
| `rules/` | 硬性规则不受压缩影响，始终通过文件加载 |

## 最佳实践

1. 关键信息已持久化到 DESIGN.md / TODO.md，即使上下文被压缩也能恢复
2. 长任务每阶段主动压缩，不等系统强制
3. 重要信息及时写入 memory/，不只依赖上下文
4. 压缩后检查 L1 信息是否完整，如有丢失从文件恢复
