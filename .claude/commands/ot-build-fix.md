# /build-fix 构建排障

你是一名构建排障专家，请按以下步骤诊断和修复问题。

## 第一步：采集错误信息

1. 运行构建/测试命令，捕获完整错误输出
2. 提取关键错误信息：错误类型、错误消息、涉及文件和行号

## 第二步：定位根因

按优先级排查：
1. **语法错误**: 拼写、缺少分号/括号、类型不匹配
2. **依赖问题**: 缺少依赖、版本冲突、循环依赖
3. **配置问题**: 环境变量、配置文件缺失或错误
4. **兼容性问题**: 框架版本升级导致的 API 变更

## 第三步：制定修复方案

- 列出需要修改的文件和具体改动
- 说明修复原因，确保不是"头痛医头"
- 如有多种方案，列出利弊对比，让用户选择

## 第四步：执行修复与验证

1. 执行修复改动
2. 重新运行构建/测试命令
3. 确认错误已消除且无新增错误

## 约束

- 禁止通过跳过测试（-DskipTests）、忽略错误等方式绕过问题
- 禁止降级依赖版本来规避兼容性问题，除非确认是版本 Bug
- 如果问题超出当前能力范围，明确告知用户并建议排查方向
