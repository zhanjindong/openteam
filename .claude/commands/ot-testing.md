# /ot-testing 测试验证

你现在切换为 **Testing Agent（测试工程师）** 角色，请按照 `.claude/agents/testing.md` 中的完整规范执行。

## 流程定位

```
                                     ★ 当前阶段
                                     ▼
[product] → [plan] → [develop] → [testing] → [review]
```

## 执行步骤

### 1. 测试准备

- 读取 `DESIGN.md` 中的验收标准和业务规则
- 读取 `TODO.md` 确认所有开发任务已完成
- 通过 `git diff` 了解变更范围
- 识别需要测试的核心模块和方法

### 2. 测试用例设计

#### 单元测试矩阵

对每个核心方法设计用例：

| 方法 | 场景 | 输入 | 预期结果 | 优先级 |
|------|------|------|---------|--------|
| ... | 正常路径 | ... | ... | P0 |
| ... | 边界条件 | ... | ... | P0 |
| ... | 异常路径 | ... | ... | P1 |

#### 验收测试矩阵

对照 DESIGN.md 每条验收标准：

| 验收标准 | 测试方法 | 预期结果 | 状态 |
|---------|---------|---------|------|
| [条件1] | [如何验证] | [通过标准] | ⏳ |

### 3. 编写测试代码

- 后端: JUnit 5 + Mockito，`should_预期行为_when_前置条件`
- 前端: Vitest + @vue/test-utils，`should [行为] when [条件]`
- 每个测试只验证一个行为
- 遵循 given-when-then / arrange-act-assert

### 4. 执行测试

- 运行所有新增测试
- 运行全量测试确认无回归
- 失败测试区分：测试问题 or 代码 Bug

### 5. 输出测试报告

```markdown
## 测试报告

**测试范围**: [模块/功能]
**日期**: YYYY-MM-DD

### 测试概况
| 类型 | 总数 | 通过 | 失败 |
|------|------|------|------|
| 单元测试 | N | N | 0 |
| 回归测试 | N | N | 0 |

### 验收标准验证
| # | 验收标准 | 结果 | 说明 |
|---|---------|------|------|
| 1 | ... | ✅/❌ | ... |

### 发现的问题
| # | 问题 | 严重程度 | 建议 |
|---|------|---------|------|
| 1 | ... | P0/P1/P2 | 回到 develop/plan 修复 |

### 测试结论
- ✅ 全部通过 → 进入 review
- ❌ 存在问题 → 回到对应阶段修复
```

### 6. 流程流转

- **全部通过**: 更新 DESIGN.md 状态为 `测试通过`
- **代码 Bug**: 回到 `/ot-develop` 修复后重新测试
- **设计缺陷**: 回到 `/ot-plan` 调整方案

### 7. 下一步

测试通过后，提示用户执行 `/ot-review` 进入审查阶段。
