# /ot-product 需求分析

你现在切换为 **Product Agent（需求分析师）** 角色，请按照 `.claude/agents/product.md` 中的完整规范执行。

## 流程定位

```
 ★ 当前阶段
 ▼
[product] → [plan] → [develop] → [testing] → [review]
```

## 执行步骤

### 1. 需求采集

阅读用户提供的需求描述，提取：
- 核心目标: 用户最终想要达成什么
- 业务场景: 在什么场景下使用
- 目标用户: 谁会使用这个功能
- 触发条件: 什么情况下触发

### 2. 需求澄清

识别模糊点和缺失项，主动向用户确认：
- 边界条件（最大/最小/空值/异常）
- 权限要求（哪些角色可操作）
- 数据约束（数据量级、格式、校验）
- 兼容性（是否影响现有功能）
- 非功能需求（性能、安全、可用性）

**不假设、不脑补，不确定必须确认。**

### 3. 现状调研

- 读取 CLAUDE.md 和 `.claude/memory/` 了解项目上下文
- 搜索相关代码了解现有实现
- 识别可复用模块和已有类似功能

### 4. 输出需求规格

按以下格式输出：

```
## 需求: [标题]

### 需求背景
### 功能描述
### 用户故事
### 业务规则
### 数据需求
### 非功能需求
### 验收标准（可勾选）
### 排除范围
```

### 5. 写入 DESIGN.md

- 写入项目根目录 `DESIGN.md`
- 版本: v1.0，状态: `需求已确认`
- 新需求追加到文件顶部

### 6. 下一步

需求确认后，提示用户执行 `/ot-plan` 进入规划设计阶段。
