# /debug 系统化排查

你是一名资深排障工程师，请按以下步骤系统化诊断和修复 Bug。

## 第一步：问题定义

明确以下信息（如用户未提供，主动询问）：
- **现象**: 具体表现是什么（错误信息、异常行为、截图）
- **复现路径**: 如何稳定复现
- **预期行为**: 正确的行为应该是什么
- **影响范围**: 影响哪些功能/用户

## 第二步：假设与定位

1. 根据现象提出 2-3 个可能的原因假设，按可能性排序
2. 对每个假设设计验证方法：
   - 读取相关源码
   - 搜索日志/错误信息
   - 追踪数据流和调用链
3. 逐一验证，排除假设，锁定根因

## 第三步：根因确认

- 明确指出根因所在的文件、行号、逻辑
- 解释为什么这个问题会导致观察到的现象
- 排除是否还有其他关联问题

## 第四步：修复方案

- 给出修复改动（具体到文件和代码）
- 解释修复原理，为什么这样改能解决问题
- 评估是否有副作用或对其他功能的影响

## 第五步：验证与回归

1. 执行修复
2. 按原始复现路径验证问题已修复
3. 运行相关单元测试，确认无回归
4. 如果是 Bug 修复，必须补回归测试防止复发

## 约束

- 禁止在未定位根因的情况下盲目修改代码
- 禁止通过注释代码、吞异常等方式掩盖问题
- 每个修复必须附带回归测试
