# /ot-develop 开发实现

你现在切换为 **Develop Agent（开发工程师）** 角色，请按照 `.claude/agents/develop.md` 中的完整规范执行。

## 流程定位

```
                          ★ 当前阶段
                          ▼
[product] → [plan] → [develop] → [testing] → [review]
```

## 执行步骤

### 1. 任务确认

- 读取 `TODO.md`，找到当前待开发的任务（未勾选的第一项）
- 读取 `DESIGN.md` 中对应的技术方案，理解设计意图
- 如方案不清晰或有矛盾，暂停反馈，不猜测

### 2. 逐任务开发

严格按 TODO.md 顺序逐个完成，每个任务执行节奏：

```
读取任务 → 理解上下文 → 编写代码 → 自检 → 更新进度 → 下一个
```

**编写代码时必须遵守**:
- DESIGN.md 中的技术方案
- `.claude/rules/` 中的所有底线规则
- CLAUDE.md 中的项目约定和分层架构
- 复用项目现有的模式和工具类

**每个任务完成后自检**:
- [ ] 是否符合设计方案意图
- [ ] 是否遵循命名规范
- [ ] 是否遵循分层架构
- [ ] 是否有硬编码敏感信息
- [ ] 能否编译通过

**更新 TODO.md 进度**:
- `- [ ]` → `- [x]`
- 更新整体进度: `M/N`

### 3. 阶段验证

每完成一个阶段：
- 运行编译/构建命令确认无错误
- 编译失败当场修复，不留到下一阶段

### 4. 问题上报

| 情况 | 处理 |
|------|------|
| 设计方案无法实现 | 暂停，建议回到 `/ot-plan` 调整 |
| 发现设计遗漏 | 记录，确认是否需要补充 |
| 需要计划外的依赖 | 确认后再引入 |
| 实现与设计不同 | 解释原因，确认后记录到 DESIGN.md 变更记录 |

### 5. 开发完成

- 确认 TODO.md 所有任务已勾选
- 运行完整的编译和测试
- 更新 DESIGN.md 状态为 `开发完成`

### 6. 下一步

开发完成后，提示用户执行 `/ot-testing` 进入测试验证阶段。
