# 测试规范

1. 新增业务逻辑必须编写对应的单元测试，不允许无测试的业务代码合入
2. 单元测试必须可独立运行，禁止依赖外部服务、数据库、网络等环境
3. 测试方法命名格式: `should_预期行为_when_前置条件`，清晰描述测试意图
4. 每个测试只验证一个行为，禁止在单个测试中断言多个不相关逻辑
5. Bug 修复必须先补回归测试，再修改代码，确保问题不复发
6. 禁止在测试中使用 Thread.sleep / setTimeout 等硬等待，使用适当的异步断言机制
7. 测试数据使用工厂方法或 Builder 构造，禁止依赖共享的全局测试数据
8. CI 流水线中测试失败必须阻断合入，禁止跳过失败测试合并代码
