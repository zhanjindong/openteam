# 安全规范

1. 密钥、Token、密码等敏感信息禁止硬编码，必须通过环境变量或配置中心注入
2. 禁止将 .env、credentials、密钥文件提交到 Git 仓库
3. 用户输入必须做参数校验和转义，防止 SQL 注入、XSS、命令注入
4. API 接口除白名单外必须经过认证鉴权，禁止裸露未授权接口
5. 日志中禁止输出用户密码、Token、身份证号等敏感字段
6. 数据库操作禁止拼接 SQL，必须使用参数化查询或 ORM 框架
7. 文件上传必须校验类型、大小，存储路径禁止由用户输入控制
8. 第三方依赖引入前必须确认无已知高危漏洞
