# OpenTeam — AI 协作指南

## 会话初始化

每次会话启动时：
1. 如果 `.claude/memory/` 目录存在，先读取其中所有 .md 文件，特别关注 `handoff.md` 了解上次交接状态（当前进展、关键决策、注意事项、下一步）
2. 如果 `TODO.md` 存在，读取当前任务进度，从上次中断的位置继续
3. 如果 `DESIGN.md` 存在，了解当前需求的设计方案

## 上下文压缩

当对话过长或上下文接近限制时，按照 `.claude/hooks/strategic-compact.md` 中的压缩协议执行结构化压缩：
- L1（核心锚点）完整保留：任务目标、设计方案、已改文件、当前进度
- L2（工作状态）压缩为摘要
- L3/L4（过程细节）丢弃
- 将有价值的发现写入 `.claude/memory/` 持久化

## 会话结束（⚠️ 必须在结束前主动执行，不要依赖 Stop hook）

当用户说"结束"、"下次继续"、"先到这"等结束信号，或你判断当前任务告一段落时，**必须在回复结束前**执行以下步骤：

1. **主动写入** `.claude/memory/handoff.md` 的以下段落（Stop hook 会写入 git 元数据，但以下内容只有你能写）：
   - `## 当前状态`：一句话总结本次会话的进展和阶段
   - `## 关键决策`：本次会话中做出的技术决策及理由（特别是被放弃的方案）
   - `## 注意事项`：发现的坑、临时方案、脆弱区域
   - `## 下一步`：接手后应该先做什么，已知阻塞或待确认问题
2. 如有新的技术决策或踩坑经验，同步更新对应的 memory 文件
3. 执行 `git add .claude/memory/`

> **为什么要主动写**：Stop hook 在你回复结束后才触发，此时你可能无法再执行写入操作。所以必须在最后一条回复中完成 handoff 写入。

## 分支规范

- `master`: 生产代码，只用于创建开发版本或 hotfix
- `release`: 生产构建基准分支
- `feature/*`: 功能开发，合入后删除
- `hotfix/*`: 线上修复，合入后删除
