# OpenTeam — AI 协作指南

## 会话初始化

每次会话启动时：
1. 如果 `.claude/memory/` 目录存在，先读取其中所有 .md 文件，特别关注 `handoff.md` 了解上次交接状态（当前进展、关键决策、注意事项、下一步）
2. 如果 `TODO.md` 存在，读取当前任务进度，从上次中断的位置继续
3. 如果 `DESIGN.md` 存在，了解当前需求的设计方案

## 上下文压缩

当对话过长或上下文接近限制时，按照 `.claude/hooks/strategic-compact.md` 中的压缩协议执行结构化压缩：
- L1（核心锚点）完整保留：任务目标、设计方案、已改文件、当前进度
- L2（工作状态）压缩为摘要
- L3/L4（过程细节）丢弃
- 将有价值的发现写入 `.claude/memory/` 持久化

## 记忆持久化（三级触发 + Stop hook 兜底）

handoff.md 采用"里程碑增量写入 + 会话结束兜底 + Stop hook 自动补充 git 数据"架构。

### 触发级别

| 级别 | 触发条件 | 写入要求 |
|------|---------|---------|
| 里程碑 | 功能完成、重要决策、踩坑解决 | **必须写入** |
| 会话结束 | 用户结束信号 / 任务告一段落 | **必须写入**（兜底） |
| 增量 | 每 3 次 commit / 上下文压缩时 | 建议写入 |

### 写入分工

| 段落 | 谁写 | 时机 |
|------|------|------|
| 元数据（时间/用户/分支） | Stop hook | 每次会话结束 |
| 本次改动 | Stop hook | 每次会话结束 |
| 最近提交 | Stop hook | 每次会话结束 |
| 当前状态 | Claude | 里程碑 + 会话结束 |
| 关键决策 | Claude | 做决策时**追加** |
| 注意事项 | Claude | 发现问题时**追加** |
| 下一步 | Claude | 里程碑 + 会话结束 |

### 写入原则

- **当前状态** 和 **下一步**：直接替换为最新内容
- **关键决策** 和 **注意事项**：追加不覆盖，保留历史记录
- git commit 后会收到 PostToolUse hook 提醒，根据提交类型判断是否需要更新
- 会话结束前**必须在最后一条回复中**完成 handoff 写入（Stop hook 只更新 git 段落）
- 如有新的技术决策或踩坑经验，同步更新对应的 memory 文件
- 写入后执行 `git add .claude/memory/`
